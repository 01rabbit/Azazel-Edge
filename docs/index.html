Operator-first design

Local by default

Offline-capable architecture with deterministic rules, where AI is advisory only and never the sole decision maker.

オフライン運用を前提とし、AI は補助に限定し、判断はすべて決定論エンジンが行います。

What it does

Azazel-Edge delivers a compact SOC/NOC gateway for emergency use. It runs on Raspberry Pi, visualizes internal network traffic, evaluates threat state, and supports operator workflows.

Azazel-Edge は、緊急運用向けのコンパクトな SOC/NOC ゲートウェイです。Raspberry Pi で動作し、内部ネットワーク、脅威判定、運用ワークフローを提供します。

Approach

Deterministic evaluation is the primary path. AI is used only as advisory support, ensuring decisions remain transparent and auditable.

決定論評価が主体です。AI は補助としてのみ使われ、判断は透明かつ監査可能に保たれます。

Key features

Offline first

Minimize cloud dependency and run core capabilities entirely on-premises for field or event deployments.

クラウド依存を最小化し、フィールドやイベント現場でローカルだけで動かせます。

Deterministic logic

NOC and SOC evaluations are fixed-rule based, producing one of five clear actions: observe, notify, throttle, redirect, isolate.

NOC/SOC の評価はルールベースで、observe／notify／throttle／redirect／isolate の 5 つの行動を明確に出します。

AI advisory

Local Ollama integration generates advice, summaries, and candidate insights without letting AI act unilaterally.

Ollama と連携して助言や要約を生成し、AI が単独で行動することはありません。

Runbook workflow

Operator guidance and controlled execution are backed by structured runbooks for fast, repeatable incident response.

構造化されたランブックで、迅速かつ再現性の高い対応を支援します。

Architecture

Evidence Plane

Normalizes events from sources like Suricata EVE and feeds them into the deterministic evaluation flow.

Suricata EVE などのイベントを正規化し、決定論評価に渡します。

NOC / SOC Evaluator

Separately assesses network and security state, producing a traceable evaluation for the arbiter.

ネットワークとセキュリティを別々に評価し、アービタに渡す判定を生成します。

Action Arbiter

Selects a single action with supporting rationale and logs the decision for audit and operator review.

サポート理由とともにアクションを選択し、監査・レビュー用に記録します。
