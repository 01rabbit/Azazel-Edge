# Azazel-Edge 一枚紙

## 現場のネットワークに、判断できる小さな防御拠点を置く

Azazel-Edge は、災害対応、臨時拠点、イベント会場、小規模事業所など、十分な監視設備や専門人員をすぐに用意できない場所で使うための、Raspberry Pi 向けエッジ型 SOC/NOC 支援ゲートウェイです。

専門的に言えば、ネットワークの異常や攻撃兆候をローカルで観測し、通信の安定性とセキュリティ上の危険を分けて評価し、取るべき行動を説明つきで提示・実行する装置です。

もっと簡単に言えば、現場のネットワークに置く「見張り役兼交通整理役」です。危ない通信を見つけたとき、ただ警告するだけでなく、観測、通知、遅延、誘導、隔離といった限定された対応を選び、なぜそうしたのかを記録します。

## 何がうれしいのか

| 現場の困りごと | Azazel-Edge が助けること |
|---|---|
| 専門家が常駐していない | 画面と通知で、今の状態と次に見るべき点を示す |
| インターネットやクラウドに頼れない | 現場内で観測、判断、記録を行う |
| 攻撃なのか障害なのか分かりにくい | 通信の安定性と脅威兆候を分けて整理する |
| 急いで止めると業務まで止まりそう | 遮断だけでなく、遅延・通知・誘導など段階的に対応する |
| 後から説明できない対応は困る | 選んだ対応、選ばなかった対応、根拠を監査ログに残す |

## ユースケース

### 1. 避難所・臨時医療拠点

避難所や臨時医療拠点では、ネットワークが急造され、利用者も機器も短時間で増えます。専門の監視チームが常駐しない場合、障害と攻撃の見分けが難しくなります。

Azazel-Edge を入口に置くことで、異常な通信をローカルで検知し、緊急度に応じて通知、通信制御、隔離を行えます。重要なのは、現場の運用をすぐ止めるのではなく、業務継続と安全確保の間で段階的に判断できる点です。

### 2. イベント会場・展示ブース・一時ネットワーク

カンファレンス、展示会、訓練会場では、多数の端末が短時間だけ同じネットワークを使います。持ち込み端末、デモ機材、来場者用ネットワークが混在し、誰が何をしているかを細かく追いにくくなります。

Azazel-Edge は、会場ネットワークの手前で怪しい通信を観測し、必要に応じて遅延や囮への誘導を行います。攻撃を完全に消す装置ではありませんが、現場担当者が「今、何が起きているか」を早く把握し、慌てて全停止する前に判断する時間を作ります。

### 3. 小規模事業所・出張所

小規模拠点では、大規模な SIEM や SOC を導入しにくく、問題が起きても本部や外部ベンダーに連絡するまで状況が分からないことがあります。

Azazel-Edge を拠点側に置くと、最低限の観測、状態表示、通知、監査記録をローカルで行えます。本部側に送る前に、現場で要点を整理できるため、問い合わせや初動対応が短くなります。

### 4. サイバー訓練・教育

訓練では、攻撃を受けたときの検知、判断、説明、記録を短時間で見せる必要があります。しかし、一般的なセキュリティ製品は構成が重く、訓練シナリオの再現性を確保しにくい場合があります。

Azazel-Edge は、決定過程とログを見せることを重視しています。どの証拠を見て、どの対応を選び、どの対応を選ばなかったのかを確認できるため、単なるデモではなく、判断訓練の教材として使えます。

## Azazel-Edge がすること

Azazel-Edge は、現場ネットワークの通信やセキュリティイベントを取り込み、通信品質上の問題とセキュリティ上の問題を分けて評価します。そのうえで、観測、通知、遅延、誘導、隔離といった限定された対応から一つを選び、画面や通知でオペレーターに示します。

AI は補助として使えますが、最終判断の中核ではありません。要約や説明を手伝う存在であり、Azazel-Edge の基本は、再現可能で説明可能な決定ロジックです。

## Azazel-Edge がしないこと

Azazel-Edge は、すべての攻撃を防ぐ万能装置ではありません。大規模 SIEM の代替でも、自律的に戦う AI 防御システムでもありません。現場の限られた条件下で、早く観測し、段階的に対応し、後から説明できるようにするための支援装置です。

## 一言で言うと

Azazel-Edge は、専門家が少なく、時間も設備も限られた現場で、ネットワークを見張り、危険を整理し、次の一手を説明つきで示すための小型エッジ防御ゲートウェイです。
